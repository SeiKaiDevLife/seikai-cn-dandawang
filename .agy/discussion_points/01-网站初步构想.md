# 私有化宠物相册网站（类似 Instagram/小红书） - 初步构想与执行计划

## 一、 项目背景与核心目标
本项目旨在打造一个属于夫妻两人的私有化宠物照片分享平台。通过纯前端架构结合阿里云 OSS，实现一个无需后端的、类似 Instagram 和小红书体验的 Web 应用。
平台主要用于记录和分享宠物的生活点滴，支持多图文发布、丰富多样的九宫格排版以及私密的双重安全防护。

## 二、 核心特性与用户体验
1. **纯前端架构**：部署于 GitHub Pages，通过 Vercel 映射自定义域名 (`dandawang.seikai.cn`)，所有动态数据交互直接通过浏览器与阿里云 OSS 进行。
2. **移动端优先（Mobile First）**：主要针对手机等窄屏设备进行精心设计和适配，视觉体验优先级最高，同时兼顾电脑端的大屏浏览体验。
3. **双重安全验证机制**：
   - **基础访问防护**：前端单向哈希（SHA-256 + 加盐）验证，隔离大众访问。
   - **核心发布防护**：阿里云发布凭证（AK/SK）通过高级对称加密算法（AES-256-GCM）加密存储。需在界面输入二级密码解密后方可执行发布操作，从根源防止恶意篡改。
4. **多模式上传与排版展示**：
   - **普通图文（小红书模式）**：首图+标题组合，点击进入后通过左右翻页浏览照片，下方展示正文和标题。
   - **标准九宫格模式**：根据图片数量（1-9张）自动排版布局（例如：5张为上3下2，7张为上3中3下1）。超9张图片时，右下角第9张照片应用模糊特效并叠加显示 `+N` 以提示剩余数量。
   - **自定义九宫格模式**：支持对九宫格进行拖拽合并（如跨行、跨列合并相邻方块），实现高度自由的排版效果，然后对各个独立格子分别上传照片。
5. **智能瀑布流布局**：首页自动根据内容高度，将最新图文优先分配到高度最低的列中，实现无缝瀑布流浏览体验。
6. **纯净宠物相册模式**：提供一个无文案、无标题的纯享版图片时间线页面。
7. **全文与时间搜索**：顶部搜索框支持多维度（关键词+时间区间）内容过滤。

## 三、 UI/UX 布局规划
- **登录页**：高颜值、纯净的单密码输入框，拦截未授权访问。
- **首页顶部导航**：居中的搜索栏（点击展开“关键词搜索”与“时间筛选”），最右上角“发布”按钮（点击唤起二级密码验证）。
- **首页头部区**：精美的头图（Hero Image）及宠物的专属头像。
- **首页内容区**：瀑布流排版的卡片式笔记，卡片包含图片（头图或自定义九宫格）及底部标题。
- **发布与上传页**：提供“普通图文”、“九宫格”、“自定义九宫格”三种模式的选择。支持自由调整图片顺序（拖拽）、自定义合并网格，以及标题与正文的编辑。

## 四、 阶段性执行计划

### 第一阶段：UI 骨架与基础页面搭建
- 确定整体配色、高质感 UI 细节及微交互设计规范。
- 开发前端登录页（哈希密码校验及本地状态保持）。
- 搭建首页框架、顶部导航栏、搜索框和基础的静态瀑布流组件。
- 搭建无文本的“宠物相册”专用展示页面。

### 第二阶段：发布核心交互组件开发
- 开发发布页面的模式切换面板。
- 实现前端图片选择、拖拽排序交互。
- 重点研发“标准九宫格”的动态类名适配与“自定义九宫格”的单元格拖拽合并计算逻辑。

### 第三阶段：核心链路打通（前端图片处理与 OSS 桥接）
- 引入前端 WebP 压缩机制：支持读取本地文件，等比缩放至长边对齐 2160px，并导出为 WebP。
- 引入 AES-256-GCM 解密模块，实现前端二级密码换取真实 OSS 上传凭证。
- 建立 OSS 数据交互链路：下载索引 `JSON` -> 上传处理后的图片 -> 在内存中生成新条目并合并到 `JSON` -> 覆盖上传 `JSON` 索引文件。
- **引入 OSS 动态图片处理与分桶缓存策略**：在前端请求图片时计算所需渲染大小，向上取整匹配固定档位后调用 OSS 的 Resize 能力，兼顾渲染速度与缓存命中率。

### 第四阶段：本地 Python 脚本与数据 Mock 建设
- 利用已有的 `_uploader` 文件夹结构，编写 Python 脚本。
- 脚本功能：遍历本地图片，生成满足上述数据结构要求的初始测试数据（`json`）及压缩图片。将其放入 `data/` 和 `public/` 中，便于前期快速进行前端 UI 视觉与排版算法的 debug。

### 第五阶段：测试与调优
- 移动端设备真实屏幕测试（兼容性与瀑布流视觉表现）。
- 上传链路的鲁棒性测试，及加密机制的安全性推演验证。
- 瀑布流性能调优及图片懒加载处理。
