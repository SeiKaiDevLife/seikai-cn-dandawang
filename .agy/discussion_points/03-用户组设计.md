# 用户系统、互动及详情页改版方案计划书

为了满足不同用户（SeiKai & Echo）登录后的头像个性化展示、互动（收藏与评论）、安全删除以及流量优化的需求，本计划书详细阐述数据结构调整、功能逻辑以及 UI 改版设计。

---

## 一、 数据结构调整

所有的静态资源和元数据将严格区分存放路径，均以 Vue 响应式数据驱动，保证离线与 OSS 同步的兼容性。

### 1. 文章数据结构（Month Post JSON）
每个帖子将新增 `publisher`（发布者）字段和 `comments`（评论列表）字段。
```json
{
  "id": "1781512400000-abcd",
  "publisher": "seikai", // "seikai" 或 "echo"，旧帖子默认归属 "seikai"
  "timestamp": 1781512400000,
  "title": "今日风景",
  "content": "天气很好！",
  "images": [
    "images/2026-06/2026-06-16_今日风景/1.webp"
  ],
  "layout": "normal",
  "comments": [ // 评论数组，默认为空数组 []
    {
      "id": "c-1781512500000-xyz",
      "commenter": "echo", // "seikai" 或 "echo"
      "text": "赞！👍", // 支持 emoji
      "image": "images/comments/2026-06/c-1781512500000-xyz.webp", // 可选，图片评论的 OSS 相对路径
      "timestamp": 1781512500000
    }
  ]
}
```

### 2. 收藏数据结构（`favorites.json`）
在 `public/` 目录下新增 `favorites.json` 文件以存储用户的收藏记录。两个用户的收藏互不干涉。
```json
{
  "seikai": [
    "1781512400000-abcd"
  ],
  "echo": []
}
```

## 二、 静态资源组织（目录划分）

根据要求，我们将所有**需要上传到 OSS 的文件**严格保存在 `public/` 目录下，其他临时脚本或源文件保存在相应外部目录。

### 1. 需要上传至 OSS 的资源 (在 `public/` 下)
* **用户头像**：
  * `public/avatars/seikai.webp` (800x800px, 1:1, WebP)
  * `public/avatars/echo.webp` (800x800px, 1:1, WebP)
* **全局收藏数据**：
  * `public/favorites.json`
* **评论图片**：
  * `public/images/comments/YYYY-MM/c-时间戳.webp` (上传生成的评论附图，限制最大长边 800px 以节约流量)

### 2. 本地构建与源文件 (不在 `public/` 下)
* **头像源文件**：
  * `_uploader/image/seikai.jpg`
  * `_uploader/image/echo.jpg`
* **处理脚本**：
  * `_uploader/convert_avatars.py` (用来裁切、压缩生成 800px 1:1 WebP 头像的本地脚本)

## 三、 UI/UX 设计方案（借鉴小红书与微博）

### 1. 头像处理 (本地 Python 自动化)
在 `_uploader` 下编写 `convert_avatars.py`。
* 读取 `_uploader/image/seikai.jpg` 与 `echo.jpg`。
* 自动居中裁剪为 1:1 正方形。
* 缩放至 `800x800` px，导出为 WebP 格式。
* 输出路径为 `public/avatars/seikai.webp` 及 `public/avatars/echo.webp`。

### 2. 详情页（Detail View）布局重构
采用小红书式的精细卡片流与双栏固定交互结构：

* **顶部固定导航栏 (`.detail-header`)**：
  * **背景**：毛玻璃半透明背景（`backdrop-filter: blur(10px)`）搭配细微阴影。
  * **左侧**：简洁优雅的返回按钮（`<`），点击关闭详情。
  * **中间**：发布者圆形头像（`32px`，带浅金/淡雅边框）和发布者昵称（`SeiKai` 或 `Echo`）。
  * **右侧**：三点操作菜单（`...`）。点击展开下拉面板（`.detail-actions-dropdown`）：
    * 如果当前登录用户是**该文章的发布者**，则显示“删除此图文”选项。
    * 否则置灰或隐藏删除选项（保证权限隔离）。

* **底部固定操作栏 (`.detail-footer`)**：
  * **输入框**：扁平化药丸形状的输入框，默认提示“说点什么...”。
  * **右侧功能区**：
    * **`+` 图片图标**：点击选择本地图片用于评论。选择后，会在底部操作栏上方悬浮一个极窄的预览卡片（`.comment-img-preview`），展示待发送的图片缩略图，并附带一个小小的 `×` 删除图标，允许随时撤销图片上传。
    * **`⭐` 收藏星标**：如果是未收藏状态，显示空心星（`☆`）；已收藏状态则显示高亮黄色的实心星（`★`），带有点击微动动画。
  * **发送按钮**：当输入框有文字或已选择图片时，右侧划出“发布”文字按钮。点击触发评论保存。

* **评论展示区 (`.detail-comments-section`)**：
  * 位于图文正文下方。
  * 列表按时间正序排列。
  * 每一个评论包含：
    * 评论者头像（圆形，`28px`，点击可查看大图）。
    * 评论者名字、时间标签。
    * 评论内容文本（全面支持 Emoji）。
    * 如果有附加图片，则在文字下方展示圆角缩略图（限制最大高度 `120px`，点击可以在新图层中放大查看）。

## 四、 核心功能与安全逻辑

### 1. 二级密码会话免输入机制（Session Memory Cache）
* **逻辑**：
  1. 在 `setup()` 中维护一个局部内存变量 `sessionSecPassword = ref('')`（纯 JS 变量，不写入 `localStorage` / `sessionStorage`，标签页关闭或刷新后即彻底消失，保证安全）。
  2. 凡是涉及 OSS 修改的操作（发布新图文、删除图文、添加评论、上传评论图、切换收藏状态）：
     - **检查**：若 `sessionSecPassword.value` 已存在，直接使用其解密 `ENCRYPTED_CREDENTIALS` 并自动初始化 OSS 客户端执行操作，免去弹窗。
     - **弹窗**：若为空，则弹出二级密码验证输入框，验证成功后将密码保存在 `sessionSecPassword.value` 中，再执行后续动作。

### 2. 评论与图片评论上传逻辑
1. 用户选定评论图片后，点击“发布”。
2. 若有图片，先利用 Canvas 压缩成最大长边 `800px` 的 WebP Blob，再直连上传至 OSS 的 `dandawang/public/images/comments/YYYY-MM/` 下。
3. 组合评论文本及图片相对路径，生成 comment 对象。
4. 拉取最新的本月文章 JSON，定位到该 post 的 `id`，将评论 `push` 到其 `comments` 列表中。
5. 将更新后的本月文章 JSON 重新 PUT 覆盖上传到 OSS，同步更新前端数据状态。

### 3. 删除逻辑
1. 用户点击“删除此图文”。
2. 系统弹出二次确认框，并验证/读取二级密码。
3. **OSS 级删除**：
   - 提取该帖子中所有图片的 OSS 路径，依次调用 `ossClient.delete()` 予以物理删除（节约 OSS 空间）。
   - 如果评论中包含图片，亦一并删除对应的评论图片。
4. **元数据合并更新**：
   - 从本月文章 JSON 中过滤移除此帖子，将更新后的 JSON 写入 OSS。
   - 在 `meta.json` 中，更新本月帖子计数（`postCount` 减 1，`totalPosts` 减 1）。如果本月帖子数归零，则从 `months` 数组中移除该月份节点，并物理删除该月份 JSON 文件。
   - 重新 PUT `meta.json` 至 OSS。
5. 刷新前端的瀑布流与状态，关闭详情页。

### 4. 收藏逻辑
1. 点击“收藏”星标。
2. 触发会话验证（读取/要求输入二级密码）。
3. 拉取最新的 `public/favorites.json`。
4. 根据当前登录者（`seikai` 或 `echo`），在对应的数组中加入或移除该图文 `id`。
5. 覆盖上传 `favorites.json` 至 OSS，同步更新前端的收藏状态。
